## Memory Maintenance

When adding entries:

1. Add a brief summary to **Recent Patterns** or **Recent Failures** (one to three lines with pattern_type and component tags).
2. Add the full entry to the appropriate section below (Pattern Tracking, Failure Tracking, etc.).
3. If Recent Patterns or Recent Failures exceeds 5 entries, remove the oldest entry from the short list (the full entry remains in the section below).

When reading memory at session start: read Recent Patterns and Recent Failures first. Dive into the full sections only if the task component or domain matches entries there.
