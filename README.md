# FORGE

FORGE is a skills-first governance workflow for AI-assisted coding.

The canonical skill pack lives in [`skills/forge/`](./skills/forge/). This repo also includes install scripts, verification helpers, optional CI enforcement, tool-scaffolding scripts, and philosophy docs.

## Quick Start

Install the skill pack:

```bash
bash install.sh
```

Then in a project:

```text
Use the forge skill and bootstrap docs/forge for this project.
```

If the project already has `docs/forge/`:

```text
Use the forge skill and execute the next bounded task from docs/forge.
```

See [GETTING_STARTED.md](./GETTING_STARTED.md) for the fuller flow and [MIGRATION.md](./MIGRATION.md) for upgrading older FORGE repos.

## Repo Layout

- `skills/forge/` - canonical FORGE skill pack
- `install.sh`, `install.ps1`, `uninstall.sh`, `uninstall.ps1` - install helpers
- `verify-install.sh`, `verify-repo.py` - verification helpers
- `ci/` - optional governance validators and hooks for team mode
- `scripts/` - tool-development scaffolding and related helpers
- `philosophy/` - design rationale and background material

Reusable agent/editor onboarding files live in `skills/forge/assets/agent-surfaces/`.
Repo-specific operational files such as GitHub workflows stay at the root where the platform expects them.

## Workflow Model

FORGE treats:

- skills as the stable execution contract
- `docs/forge/` as project-local state
- CI and hooks as optional external enforcement

In solo mode, the agent should finish one task, update `TASKS.yaml`, create a Conventional Commit, and stop before moving on.

In team mode, FORGE supports:

- task claiming on `forge-state`
- feature branches for implementation
- an integration branch such as `develop`
- a release branch such as `main`
- per-task checkpoints and Conventional Commits
- optional CI enforcement from `ci/`

Commit history should describe the work only and must not include AI attribution such as `Generated by Claude` or `Coded with Cursor`.

## Tool Repos

To scaffold a new tool repository, use:

- `scripts/forge-tool-init.sh`
- `scripts/forge-tool-init.ps1`

These scripts seed minimal `docs/forge/` files from the canonical skill assets and point the new repo at the skills-based workflow.

## Verification

To verify the repo and install flow:

```bash
python3 verify-repo.py
```
